# 更新日志

本文件用于记录每个正式版本的更新内容。GitHub Actions 发布 `v*` 标签时，会读取对应版本小节并写入 GitHub Release 和 `latest.json.notes`，应用内“检查更新”弹窗也会展示这里的内容。

## [1.1.0] - 2026-04-22

### 新增

- 主界面重构为专业级三栏工作台：顶部上下文 Header、左侧项目与模块区、中间构建配置区、右侧执行输出区。
- 中间配置区新增 `执行配置 / 环境检测 / 高级参数` Tabs，并新增高级参数面板，支持本地仓库、revision、并行线程和追加 Maven 参数。
- 右侧输出区新增 `日志 / 历史 / 模板` Tabs，日志区支持搜索、复制、清空、停止、放大和自动滚动控制。
- 左侧新增独立 Git 状态卡片和常用组合卡片，常用组合支持保存当前配置并一键应用。

### 优化

- 构建摘要卡片突出显示当前状态、目标模块、耗时和主操作按钮。
- 模块树支持短名称显示、完整 artifactId 悬浮提示、清空选择、展开全部、收起全部和仅显示已选模块。
- 常用组合卡片固定高度并支持内部滚动，避免模板数量增加后撑开左侧布局。
- 中间配置区 Tab 栏固定显示，仅内容区域滚动，减少配置时的上下查找成本。
- Git 本地未提交改动提示改为紧凑文案，减少左侧空间占用。
- 统一卡片圆角、边框、间距和浅色视觉风格，提升桌面工具完成度。

### 修复

- 修复左侧常用组合卡片底部边缘与主工作区底部不齐的问题。
- 修复常用组合模块信息过长导致列表撑宽的问题，现仅展示组合名称和应用按钮。

## [1.0.4] - 2026-04-22

### 新增

- 选择 Git 项目后展示当前分支和本地 Git 状态，避免自动联网触发 Git 凭据弹窗。
- 支持手动执行 `git fetch` 检查远端未拉取提交。
- 支持在应用内执行 `git pull --ff-only`，只允许干净更新，不允许自动合并,并提示用户优先在代码编辑器中拉取以便处理冲突。
- 支持展示当前 Git 分支，并可在本地分支之间切换。
- 应用内 Git 命令默认使用 Git Credential Manager，减少 CredentialHelperSelector 弹窗。
- 优化主界面为局部滚动布局，并压缩环境识别模块的信息展示。
- 将命令预览移动到工作区首位，提升高频打包操作的可见性。
