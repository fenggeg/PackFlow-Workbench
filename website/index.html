Why PackFlow

从构建到运维，不再在终端之间反复切换。

多模块项目的痛点不只是打包——环境配置、产物定位、远程部署、服务重启、日志排查散落在不同工具里。PackFlow 把它们放到同一个操作闭环中。

Build Center

构建命令可视化编排

自动解析 Maven 多模块结构，集中管理 JDK、Maven、Profile 和常用参数，执行前可预览并微调最终命令，构建失败自动诊断原因。

Artifact Manager

构建产物一站式管理

聚合所有模块的构建产物，按模块和规则筛选可部署文件，支持打开文件夹、复制路径、一键部署。

Command Center

命令调度与部署流水线

配置命令模板和步骤链，支持 SSH 命令执行、文件上传和等待步骤，模板变量灵活配置，后台命令支持断开日志连接。

Server Management

服务器全功能运维

管理 SSH 连接和分组，内置远程终端、文件浏览上传下载、日志实时查看和常用命令快捷执行，密码通过 DPAPI 加密保存。

Service Ops

服务监控与运维

维护服务运行时配置，一键重启并自动健康检查，远程日志实时流支持关键字过滤，操作历史可追溯。

Windows Native

为本地桌面场景打磨

基于 Tauri 2 与 Rust 后端，结合 SQLite 持久化、DPAPI 加密、SSH 双引擎和 Job Objects 进程管理，贴近 Windows 运维习惯。

FAQ

安装前最关键的问题。

它是 Web 系统还是桌面应用？

它是面向 Windows 的本地桌面应用，用来管理本地 Maven 项目的构建、部署和服务器运维。

支持哪些部署步骤？

当前支持 SSH 命令执行、文件上传和等待步骤。命令步骤支持变量展开、超时设置和特权模式。

能远程管理服务器吗？

可以。内置远程终端（xterm.js + SSH PTY）、文件浏览与上传下载、日志实时查看和常用命令快捷执行，无需额外 SSH 客户端。

数据存在哪里？

项目、构建历史、服务器、服务配置和操作记录保存在应用数据目录中的 SQLite 数据库，按项目隔离。

服务器密码如何保存？

Windows 下通过 DPAPI 加密保存，不以明文落盘。

SSH 连接依赖系统 ssh.exe 吗？

不依赖。应用内置 ssh-rs 和 ssh2 双引擎，独立处理 SSH 连接、SFTP 传输和远程命令执行。
